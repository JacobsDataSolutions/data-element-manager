# Contributing

Thank you for taking an interest in **Data Element Manager**! This project is primarily a reference architecture and learning resource.

However, we value diverse and differing insights from people who are as passionate about learning and continuous development as we are.

With that in mind, we welcome commentary and contributions to the code base as long as they are in accordance with the [Contributor Code of Conduct][code-of-conduct]. By participating in this project you agree to abide by its terms.

Contributions to this project are [released](https://help.github.com/articles/github-terms-of-service/#6-contributions-under-repository-license) to the public under the [project's open source license][license].

## Submitting a Pull Request

This repository uses the fork-and-pull model. Please fork the repository and submit pull requests from your fork, as described below.

0. [Fork][fork] and clone the repository

0. Set up a local build

0. Create a new branch: `git checkout -b my-branch-name`

0. Make your change

0. Push to your fork and [submit a pull request][pr]

## Code Style and Principles

- Follow existing [conventions][conventions]

- Favor maintainability and understandability over cleverness

- Aim for canonical code expression as per the latest technology version guidance, i.e.
    - [C# 14][csharp-14]    
    - [F# 10][fsharp-10]
    - [.NET 10][dotnet-10]
